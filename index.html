A vector database stores data as high-dimensional numerical vectors, allowing similarity search instead of exact keyword matching.

Unlike traditional databases that match exact values, vector DBs find "nearest neighbors" — items that are semantically similar even if they share no keywords.

How it works: Each piece of content (text, image, audio) is converted to a vector via an embedding model. The database indexes these vectors using algorithms like HNSW or IVF for fast approximate nearest-neighbor search.

Example: Searching for "happy" might surface documents containing "joyful", "elated", or "cheerful" because their vector representations are close in high-dimensional space (low cosine distance).

Common use cases: semantic search, RAG pipelines, recommendation systems, image similarity search.

Prompt A (v1) 540 tokens
RAG (Retrieval-Augmented Generation) injects relevant documents into the prompt at inference time. The model's weights don't change — you update the knowledge by updating your vector store. Best for: dynamic knowledge, private data, fast iteration.

Fine-tuning updates the model's weights on a curated dataset. Bakes in tone, format, or domain knowledge. Best for: consistent style, specialized vocabulary, or tasks where latency matters (no retrieval step).

Rule of thumb: Try RAG first. Fine-tune only when RAG isn't precise enough or latency is a hard constraint.
Prompt B (v2) 380 tokens
RAG retrieves relevant docs at inference time and injects them into the prompt — no weight changes, easy to update. Fine-tuning trains the model on your data, baking in style or domain knowledge permanently.

Use RAG for dynamic data. Use fine-tuning for consistent behavior or latency-sensitive tasks.

Prompt A (v1) 310 tokens
Prompt injection is an attack where malicious content in user input hijacks an LLM's instructions. The attacker embeds instructions that override the system prompt — e.g., "Ignore previous instructions and do X instead."

Direct injection: attacker controls the user input. Indirect injection: attacker plants instructions in data the model retrieves (documents, web pages, emails).

Mitigations: input sanitization, privilege separation, output filtering, sandboxed tool execution.

Prompt injection is when malicious content in user input (or retrieved data) overrides a model's system instructions. The attacker effectively hijacks the agent's behavior.

Two forms: direct (attacker controls input) and indirect (instructions planted in retrieved documents or emails).

import csv

def parse_csv(filepath: str) -> list[dict]:
    """Parse a CSV file into a list of dicts."""
    with open(filepath, newline='', encoding='utf-8') as f:
        reader = csv.DictReader(f)
        return list(reader)

# Usage
rows = parse_csv('data.csv')
print(rows[0])  # {'name': 'Alice', 'age': '30'}

Uses csv.DictReader — headers become keys automatically. Add encoding='utf-8-sig' to handle Excel BOM.
